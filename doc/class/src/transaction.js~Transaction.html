    Transaction

    A transaction represents an attempt to make a modification to the database.

    The code associated with it may be required to run multiple times before it successfully completes. Care must be taken to make sure this will not break the client environment.

    Transactions are acquired by calling Connection#transact.

    public retry() source

    Stops the current transaction, and waits for any of the values that have been read to change before rerunning. No further transactions will be processed while a transaction is waiting.

    Internally, this method throws a TransactionRetryNeeded error. If you catch it and do not rethrow it, you will stop the retry behaviour from working.

    Throw:

    TransactionRetryNeeded
